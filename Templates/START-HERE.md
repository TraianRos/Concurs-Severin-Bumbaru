# Start Here

Acest fisier este punctul de plecare pentru cineva care nu cunoaste bine web development, OOP sau organizarea unei aplicatii.

## Ce gasesti in repo

Repo-ul contine 5 demonstratii diferite. Fiecare arata aceeasi idee de baza:

- exista o interfata pe care o vede utilizatorul
- exista cod care proceseaza cererile
- exista o baza de date care pastreaza informatia

Fiecare proiect foloseste alte tehnologii, ca sa poti compara stilurile de lucru.

## Cu ce proiect sa incepi

- Daca vrei ceva simplu si usor de urmarit: incepe cu `01-mini-library-php`.
- Daca preferi Python: incepe cu `03-event-hub-flask`.
- Daca vrei un frontend mai dinamic: mergi la `02-sprint-board-react-express`.
- Daca vrei sa vezi o structura foarte explicita de server: incearca `04-room-reserve-go`.
- Daca vrei un exemplu mai special: uita-te la `05-quiz-lab-fastapi-wasm`.

## Cum citesti documentatia

Pentru fiecare proiect, ordinea buna este:

1. `BEGINNER_GUIDE.md`
2. `README.md`
3. `ARCHITECTURE.md`
4. `DATABASE.md`
5. codul propriu-zis

## Dictionar foarte scurt

- aplicatie web = program care ruleaza pe server si se foloseste din browser
- frontend = partea pe care o vede utilizatorul
- backend = partea care primeste cereri si decide ce face aplicatia
- baza de date = locul in care se pastreaza datele
- ruta = adresa din aplicatie, de exemplu `/books`
- request = cererea trimisa de browser catre server
- response = raspunsul primit de browser
